# 논준모연구소 방학 오프라인 특강 신청 웹사이트 설계

## 목표

논준모연구소의 방학 오프라인 특강을 안내하고 신청을 접수하는 Node 기반 웹앱을 만든다. 결제 시스템은 포함하지 않으며, 사용자는 신청 정보를 제출하고 관리자는 계좌이체 입금 확인 후 신청자별 입금확인 상태를 체크한다. 사용자 화면의 모집 인원 계산은 전체 신청 수가 아니라 입금확인 완료 인원만 기준으로 한다.

## 기술 선택

- 런타임: Node.js
- 서버: Express
- 저장소: SQLite 파일 DB
- 렌더링: 서버 렌더링 HTML 템플릿
- 스타일: 반응형 CSS
- 인증: 간단한 관리자 비밀번호 세션 로그인

Express와 SQLite를 사용해 설정은 가볍게 유지하면서도 강의, 신청자, 입금확인 상태를 안정적으로 연결한다.

## 사용자 화면

### 강의 목록

`/`에서 등록된 강의를 두 가지 방식으로 보여준다.

- 텍스트 리스트: 제목, 일정, 장소, 신청 상태를 빠르게 확인한다.
- 달력 보기: 강의 일정이 있는 날짜에 강의 제목을 배치한다.

각 강의 항목은 상세 페이지로 이동하는 링크다. PC에서는 목록과 달력이 넓게 보이고, 모바일에서는 세로 흐름으로 읽기 쉽게 배치한다.

### 강의 상세 및 신청

`/lectures/:id`에서 다음 정보를 표시한다.

- 강의 제목
- 강의 내용
- 강의 일정
- 장소
- 모집 정원
- 현재 입금확인 완료 인원
- 신청 가능 여부

신청 가능 여부는 입금확인 완료 인원이 모집 정원 미만이면 `신청중`, 정원 이상이면 `마감`으로 표시한다. `마감` 상태에서는 신청 폼을 노출하지 않거나 제출을 막는다.

신청 폼 입력값은 다음과 같다.

- 이름
- 연락처
- 이메일
- 신청 강의
- 입금자명

신청이 완료되면 입금 안내 메시지를 보여준다. 실제 결제 처리는 구현하지 않는다.

## 관리자 화면

### 로그인

`/admin/login`에서 관리자 비밀번호를 입력해 로그인한다. 로그인 세션이 없는 사용자는 관리자 기능에 접근할 수 없다.

### 강의 관리

`/admin`에서 강의를 등록, 수정, 삭제한다. 강의 필드는 다음과 같다.

- 강의 제목
- 강의 내용
- 강의 일정
- 장소
- 모집 정원
- 신청 마감일

삭제 시 연결된 신청 내역 처리 규칙은 단순하게 유지한다. 삭제된 강의의 신청 내역도 함께 삭제한다.

### 신청 내역 관리

관리자는 강의별 신청 내역을 확인한다. 신청자 정보와 입금자명을 확인한 뒤 신청자별 `입금확인` 체크 상태를 변경할 수 있다.

입금확인 체크가 켜진 신청만 사용자 화면의 현재 모집 인원에 반영된다. 체크를 해제하면 반영 인원에서도 빠진다.

## 데이터 모델

### lectures

- id
- title
- description
- schedule_at
- location
- capacity
- application_deadline
- created_at
- updated_at

### applications

- id
- lecture_id
- name
- phone
- email
- depositor_name
- payment_confirmed
- created_at
- updated_at

`applications.lecture_id`는 `lectures.id`를 참조한다. 강의 삭제 시 관련 신청 내역도 삭제한다.

## 상태 계산

강의별 입금확인 완료 인원은 다음 조건으로 계산한다.

```sql
COUNT(applications.id)
WHERE applications.lecture_id = lectures.id
AND applications.payment_confirmed = 1
```

신청 상태는 다음과 같이 계산한다.

- 입금확인 완료 인원 >= 모집 정원: 마감
- 입금확인 완료 인원 < 모집 정원: 신청중

신청 마감일이 지난 경우에도 신청을 막고 `마감`으로 표시한다.

## 디자인 방향

논준모연구소 특강 안내에 어울리도록 과한 장식보다 정보 신뢰도와 신청 흐름의 명확성을 우선한다.

- 흰색 배경, 짙은 잉크색 본문, 절제된 강조색
- 강의 일정, 장소, 상태가 빠르게 스캔되는 목록
- 달력에서는 날짜와 강의 제목이 겹치지 않는 안정적인 셀 구조
- 모바일에서는 강의 카드, 상세 정보, 신청 폼이 한 열로 자연스럽게 이어지는 구조
- 관리자 화면은 반복 작업이 쉬운 표와 간결한 폼 중심

## 검증 기준

- 관리자가 강의를 등록하면 사용자 목록과 달력에 표시된다.
- 강의 상세 페이지에서 입금확인 완료 인원과 신청 상태가 정확히 표시된다.
- 사용자가 신청하면 관리자 화면의 신청 내역에 저장된다.
- 관리자가 입금확인을 체크하면 사용자 화면의 완료 인원이 증가한다.
- 완료 인원이 정원에 도달하면 강의가 마감으로 표시되고 추가 신청이 막힌다.
- PC와 모바일 화면 모두 주요 정보와 버튼이 겹치지 않는다.
